# SLEB 代码阅读笔记

> **论文**: SLEB: Streamlining LLMs through Redundancy Verification and Elimination of Transformer Blocks (ICML 2024)
> **仓库**: https://github.com/jiwonsong-dev/SLEB

## 1. 核心思想

SLEB 以整个 Transformer block 为最小剪枝单位，通过**贪心搜索**逐轮移除使 loss 增加最小的 block。极简实现：不需要微调或蒸馏，纯前向推理 + 贪心搜索。

## 3. Block 冗余度衡量

SLEB **不**显式计算余弦相似度，而用**基于 loss 的指标**间接衡量：
- 临时设置 `pass_layer = True`（forward 直接返回输入，跳过 attention + MLP）
- 在校准集上计算整个模型的 cross-entropy loss
- **loss 增加越小 → 该 block 越冗余**

## 4. 贪心搜索算法

```
alive_list = [0, 1, ..., num_blocks-1]
for round i in [1, num_remove_blocks]:
    for block j in alive_list[barrier:-barrier]:
        turn_off(j)            # 临时关闭
        loss_j = get_loss()    # 计算 loss
        turn_on(j)             # 恢复
    select j* = argmin(loss_j) # 选 loss 最小的
    permanently turn_off(j*)   # 永久关闭
    removal_list.append(j*)
```

- **`early_barrier/latter_barrier`**（默认各 1）：保护首尾层不被移除
- **复杂度**：O(n × k) 次前向推理，n=总 block 数，k=要移除数
- **搜索空间逐轮递减**

## 5. On/Off 机制

**搜索期间**：`OnOff_LlamaDecoderLayer` 包装原始层
- `pass_layer = True` → forward 直接 `return hidden_states`（恒等映射）
- `pass_layer = False` → 正常计算

**搜索完成后**：`block_remove()` 物理删除
- `del model.model.layers[idx]`（从后往前删避免索引错位）
- 更新 `self_attn.layer_idx` 保证 RoPE 正确

## 6. 评估方法

- **PPL**：WikiText-2, C4 上的 perplexity
- **Zero-shot**：PiQA, WinoGrande, HellaSwag, ARC-Challenge, ARC-Easy
- **延迟**：`torch.cuda.Event` 精确计时，Prompt Processing + Token Generation

## 7. 设计特点

1. **极简**：纯前向推理 + 贪心，无训练/梯度
2. **Block 粒度**：整个 Transformer block 为单位（vs. head/neuron 级）
3. **On/Off 巧妙**：搜索期间用 flag 控制跳过，避免反复修改模型结构
4. **支持 LLaMA + OPT** 两种架构
